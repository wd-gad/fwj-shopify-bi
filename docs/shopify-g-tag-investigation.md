# Shopify G Tag Investigation

## 結論

2026-03-19 時点では、`G5`, `G7`, `G8`, `G9`, `G10`, `G11`, `G12` を

- `Novice`
- `Iron`
- `Steel`
- `Titan`

のいずれかへ、ストアデータだけで安全に対応付けることはできませんでした。

推測で埋めるのは危険です。

## なぜ危険か

調査した結果、Gタグには次の特徴がありました。

1. Gタグは `FWJカード会員` と一緒に付いているが、全員に同じように loyalty 情報があるわけではない
2. `loyalty.easy_points_attributes` の `tier_name` は、確認できた範囲ではほぼ `Default`
3. `percentage` も確認できた範囲では `1.0` に揃っており、Gタグによる差が見えない
4. `fwj_effectivedate` との対応にも規則性が見えない

つまり、Gタグは「現在のポイントランク」ではなく、

- 旧運用の区分
- 会員移行時のグループ
- 別用途の会員分類

などの可能性があります。

## 既存 loyalty 設定との関係

Gタグごとに `loyalty.easy_points_attributes` を確認したが、

- `tier_name` はほぼ `Default`
- `percentage` はほぼ `1`

であり、Gタグごとの明確な差は読み取れなかった。

## 実務上の判断

現時点では、次のどちらかで進めるのが安全です。

### A. 業務側に Gタグの意味を確認する

最も安全です。

- `G5 は Steel`
- `G11 は Titan`

のように業務定義をもらってから
[shopify-member-rank-map.json](/Users/takashiwada/Documents/CodeX/ShopifyApp/docs/shopify-member-rank-map.json)
へ反映します。

### B. 月末運用では会員ランク部分を別紙管理にする

もし確認に時間がかかる場合は、

- 競技数
- 早期条件

だけを先に自動計算し、会員ランクは別紙または個別確認で補完します。

## 推奨

この時点では `shopify-member-rank-map.json` を空欄のまま維持し、
業務側確認が取れたタグだけ埋める方が安全です。
